(Deprecated) Payment Instruments Create With Till Payments attributes API

The response will be a response with type 'payment-instrument-create-responses' with either an attribute `iframe_challenge_url` which has the URL to display in an iframe to contain the banks 3ds challeng, or a relationship to the new `payment_instrument` if it is created (it will be included)

Payment instruments can only be created by logged in users.

POST https://api.myfoodlink.com/smart_shopping/v2/payment_instruments

Parameters

Name Description
service_id The id of the payment service to create the instrument for
till_transaction_token The till transaction token returned from the Till js
till_3ds_success_url The URL to instruct the 3DS (or other challenge) to redirect to on success
till_3ds_cancel_url The URL to instruct the 3DS (or other challenge) to redirect to if the user cancels the challenge
till_3ds_error_url The URL to instruct the 3DS (or other challenge) to redirect to if the challenge is not completed
capabilities Capabilities of the app in regards to card tokenisation. THis is sent as an array of strings. Supported values: three_d_secure_tokenisation, three_d_secure_transaction, and zen_gift_cards
