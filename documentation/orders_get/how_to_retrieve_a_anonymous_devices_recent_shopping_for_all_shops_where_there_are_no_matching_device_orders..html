Orders Get API

Returns a set of the current user identity's (or anonymous device's) orders. If user identity is signed in only their orders across all shops are returned. If no user identity is signed in then anonymous orders associated with the device_id will be returned. For anonymous requests the state parameter is ignored - only shopping orders will be returned.

How to retrieve a anonymous devices recent shopping for all shops where there are no matching device orders.

GET https://api.myfoodlink.com/smart_shopping/v2/orders

The device id in the request does no match any shopping orders

Parameters

Name Description
state If set to 'shopping', only shopping orders will be returned. Other values will return all orders. This always defaults to 'shopping' for anonymous sessions.
shop_id If supplied, only orders from the shop will be supplied
page 1-indexed page number of page to return
page_size Number of lines to include on each page
